  36          <h1>Upgrader depuis une version précédente de b2evolution</h1>
  37          <p class="note">Avant de commencer, merci de noter que certaines fonctionnalités distribuées de b2evolution nécessitent que vous approuvier nos <a href="http://b2evolution.net/about/terms.html" target="_blank">Conditions d'utilisation</a> (très simples).</p>
  38          <ol class="expanded">
  39              <li>Faites une copie de sauvegarde de tous vos fichiers b2evolution. Prenez spécialement soin des fichiers que vous avez modifié, y compris <strong>/conf</strong>, <strong>/skins</strong>, <strong>/plugins</strong>, le répertoire <strong>/media</strong> ainsi que les fichiers stub, les hacks, les templates personnels...</li>
  40              <li>Vous devriez également faire une sauvegarde de votre base de données si vous le pouvez. Cette fonctionnalité devrait être proposée par votre hébergeur.</li>
  41              <li>Si vous mettez à jour depuis une version inférieure a 2.0, il est hautement recommandé d'effacer tous les anciens fichiers. Pour cela, supprimez tous les répertoires à l'exception de <strong>/media</strong>. Les répertoires à supprimer incluent  /admin, /conf, /cron, /htsrv, /inc, /install, /plugins, /skins, /skins_adm, /xmlsrv. Selon votre version, vous pouvez ne pas avoir tous ces répertoires sur votre serveur. N'oubliez pas de faire une sauvegarde avant de les supprimer.</li>
  42              <li>Téléchargez <strong>tous les nouveaux fichiers b2evolution du dossier /blogs</strong> sur votre serveur. Nous ne pouvons insister suffisament sur ce point: <strong>TOUS LES FICHIERS</strong>! Et ceci signifie que vous ne pouvez pas réutiliser les fichiers que vous avez modifiés dans /conf, /skins et /plugins si vous ne les avez pas renommés.<br /> <br />Etant donné que vous configurations personnalisées, gabarits personnalisés et plugins personnalisés sont censés porter des noms différents, ils ne seront typiquement pas écrasés. De même pour les fichiers du répertoire /media.<br /> <br />Le seul fichier auquel vous devez porter une attention particulière est <code>/blogs/_basic_config.php</code>. Vous devez conserver, recréer ou encore restaurer ce fichier depuis votre backup (voir étape 1) avant de pouvoir continuer avec la mise à jour.</li>
  43              <li>Executez le script d'installation sur votre serveur. Il est situé à l'emplacement <code>http://votresite.com/votredossierblogs/install</code>. Ensuite, suivez simplement les instructions à l'écran.<br /> <br /> Le script d'installation/mise à jour vous proposera de mettre à jour la base de donnée depuis <strong>n'importe</strong> quelle version de b2evolution vers la nouvelle.</li>
  44              <li>Vérifiez que la nouvelle installation fonctionne.</li>
  45              <li>Consultez le manuel en ligne pour obtenir des <a href="http://manual.b2evolution.net/Upgrade_from_an_older_version" target="_blank">instructions de mise à jour spécifiques</a> à votre nouvelle version de b2evolution.</li>
  46              <li>Sur les serveurs de production, il est recommandé de supprimer le sous répertoire <code>/install</code>.</li>
  47              <li>Si vous aviez des hacks ou autres personnalisations de fichiers, vous pouvez maintenant les réintroduire. Nous vous recommandons de vous procurer un outil de type diff/merge tel que <a href="http://winmerge.sourceforge.net/" target="_blank">WinMerge</a>. Ceci vous permettra de comparer aisément vos anciens fichiers (sauvegardés) avec vos nouveaux fichiers (et même des dossiers entiers en une seule opération) afin de reporter les modifications nécessaires.</li>
  48              </ol>
